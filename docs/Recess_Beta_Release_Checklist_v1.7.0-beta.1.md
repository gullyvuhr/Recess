# Recess Beta Release Checklist: v1.7.0-beta.1

## Android local builds

Prerequisites: Flutter, Android SDK/platform tools, Java 17, accepted Android SDK
licenses, and dependencies available to Gradle.

```powershell
flutter pub get
flutter build apk --debug
flutter build apk --release
flutter build appbundle --release
```

Expected artifacts:

- `build/app/outputs/flutter-apk/app-debug.apk`
- `build/app/outputs/flutter-apk/app-release.apk`
- `build/app/outputs/bundle/release/app-release.aab`

Install or upgrade a connected device without clearing local data:

```powershell
adb install -r build\app\outputs\flutter-apk\app-debug.apk
adb install -r build\app\outputs\flutter-apk\app-release.apk
```

The release variants require a local upload key. Copy
`android/key.properties.example` to ignored `android/key.properties`, point
`storeFile` at a local JKS file, and replace every placeholder. Never commit the
key, real properties, passwords, or signing output. Store and back up the upload
key outside the repository.

## Android upgrade validation

1. Install canonical v1.6.1 and configure a non-default workday, duration,
   difficulty, Bell sound, Quiet Hours, and notification preference.
2. Complete at least one Recess so History and Insight inputs exist.
3. Run `adb install -r` with the beta APK. Do not uninstall first.
4. Open Settings and verify every preference and the workday.
5. Verify History retains the completed Recess and Home can still derive an
   insight from prior facts.
6. Trigger preview, manual, and scheduled Bells and confirm their current
   behavior is unchanged.
7. On Android 12+, confirm Alarms & reminders access is allowed for Recess. For
   an existing configured beta, toggle Notifications off and on once to invoke
   the exact-alarm access flow, then reopen Recess or save the Workday to rebuild
   pending Bells.
8. Lock the phone before a Bell, leave it idle, and confirm the Bell arrives at
   its scheduled time without opening Recess.
9. Clear app data or reinstall only for notification-channel sound cache tests.

## iOS local build and TestFlight preparation

iOS work requires macOS, current Xcode compatible with the Flutter SDK, CocoaPods,
an Apple Developer team, a registered `com.recessapp.recess` App ID, signing
certificates, and provisioning profiles.

```bash
flutter pub get
flutter build ios --release --no-codesign
open ios/Runner.xcworkspace
```

In Xcode, select the Runner target, assign the team and provisioning profile,
confirm version `1.7.0-beta.1` and build `2`, run on a device, then use Product >
Archive for a signed distribution build. TestFlight upload, App Store Connect
metadata, compliance answers, and submission are intentionally outside this
sprint. Android device approval is complete; iOS signing and device validation
remain dependent on a macOS/Xcode environment.

## Identity and launch review

- App name reads `Recess` on both platforms.
- Cream launch surface has no white flash in light or dark system appearance.
- iOS launch wordmark is centered and stable across supported sizes.
- Original Recess bell mark appears on Android 12 and iOS LaunchScreen.
- Default Flutter launcher icons are replaced in every Android density, Android
  adaptive/monochrome configuration, and iOS AppIcon slot.
- No icon contains text, copyrighted third-party artwork, or an unapproved mark.

## Sound provenance audit

School Bell, Coach Whistle, and Gentle Chime are deterministic original
waveforms generated by `tool/generate_bell_sounds.py`. The script reads no audio
input. Superseded audio bytes have been overwritten and are no longer active or
packaged. No attribution or third-party redistribution right is required for the
generated assets.

Use `Recess_Release_Asset_Manifest_v1.7.0-beta.1.md` as the authoritative
asset-status record. The generated sounds passed listening approval on the
target Android device during Sprint 5.2 closeout.

## Final release gate

Run `dart format --set-exit-if-changed lib test`, `flutter analyze`, full
`flutter test`, `flutter build bundle`, debug APK, release APK, release AAB, and
`git diff --check`. Record any Java/Maven certificate-chain failure verbatim as
an environment limitation after confirming Flutter analysis/tests/bundle pass.
On macOS, add unsigned iOS compilation and signed device launch. Confirm there
are no logs, absolute local paths, secrets, keystores, test fixtures, unused
resources, or oversized artifacts staged for release.
